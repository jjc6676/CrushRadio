# Project Memory: CrushRadio

## High-Level Context
CrushRadio is a modular, extensible web application for streaming online radio and YouTube audio. The project is structured for maintainability, scalability, and ease of feature expansion.

## Key Architectural Points
- Component-based UI (React, Next.js)
- Serverless API endpoints for backend logic
- Helpers for shared logic and integrations
- Clear separation of concerns

## Product Requirements
- Play/manage radio streams
- Search/play YouTube audio
- Responsive, intuitive UI
- Extensible for new features/integrations

## Technical Stack
- React, Next.js, Node.js (API), CSS

## Current State
- Core directories and documentation initialized
- No authentication or user features yet

## Next Steps
- Implement core streaming and playback features
- Add error handling and user-friendly messages
- Plan for user features and scalability

## Lessons Learned
- Modular architecture supports rapid feature addition
- Clear documentation aids onboarding and maintenance
